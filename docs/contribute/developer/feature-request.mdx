## Before submitting a feature request

Before submitting a feature request, please:

1. Check if the feature has already been requested in our [GitHub issues](https://github.com/passbolt/passbolt_api/issues)
2. Check if the feature has been discussed in our [Community Forum](https://community.passbolt.com)
3. Consider if the feature aligns with Passbolt's [mission and values](https://www.passbolt.com/about)

## How to submit a feature request

You can submit a feature request in two ways:

### Through GitHub

1. Go to the [Passbolt API repository](https://github.com/passbolt/passbolt_api/issues)
2. Click on "New issue"
3. Select "Feature request" as the issue type
4. Fill in the template with:
   - A clear and descriptive title
   - A detailed description of the feature
   - Use cases and benefits
   - Any relevant mockups or examples

### Through the Community Forum

1. Go to the [Community Forum](https://community.passbolt.com)
2. Navigate to the "Feature Requests" category
3. Click "New Topic"
4. Fill in:
   - A clear and descriptive title
   - A detailed description of the feature
   - Use cases and benefits
   - Any relevant mockups or examples

## What happens next?

1. The Passbolt team will review your request
2. If the feature aligns with our roadmap, we'll:
   - Add it to our backlog
   - Create a GitHub issue to track progress
   - Update you on the status
3. If the feature doesn't align with our current priorities, we'll:
   - Explain why
   - Suggest alternatives if available
   - Keep the request for future consideration

## Tips for a good feature request

- Be specific about what you want to achieve
- Explain why the feature would be valuable
- Provide examples of how you would use it
- Consider potential challenges or limitations
- Be open to discussion and feedback
